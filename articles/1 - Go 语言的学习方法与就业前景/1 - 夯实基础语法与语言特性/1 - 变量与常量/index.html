Go 的变量与常量 — 爱的编程笔记

用言情小说的笔触，带你温柔而清晰地读懂 Go 里的变量与常量。

适合：大学生 · 初学者

引子 — 初见

那天的代码像初恋，一行行被你轻轻读过；变量是名字，常量是誓言。作为 Go 的初学者，你也许在花丛间徘徊，不知如何叫出那朵最美的变量。别怕，我来牵你的手。

第 1 节：什么是变量？

变量，在程序里就是一个有名字的容器，它可以装下某些东西——数字、文本、甚至是你悄悄放进来的情绪。Go 用 var 关键字声明变量：

// 声明一个整型变量 age
var age int
age = 21

// 声明并初始化
var name string = "小明"

// 短变量声明（函数内）
city := "北京"

短变量声明像是恋人间的一句俏皮话，简单、直接、在函数里最常用。正式的 var 更像是婚书，郑重其事。

第 2 节：类型是你的气质

在 Go 里，每个变量都有类型（type）。类型决定了你能给变量什么样的“礼物”。数字、字符、布尔……类型像气质，让变量显得可靠。

第 3 节：常量——不变的誓言

常量（const）在声明时就确定了，之后不能再改变。它是程序里的誓言，是你对世界的小小承诺。

const Pi = 3.14159
const Greeting string = "你好，世界"

第 4 节：细节与约定

在 Go 中，变量名以小写字母开头表示包内可见，以大写字母开头表示导出给包外使用。这像是对外的微笑或悄悄的私语。

实战演练：用代码讲故事

下面我们写一个小函数，像是一段短短的爱情独白，变量与常量交织其中：

package main

import "fmt"

const defaultGreeting = "嗨，陌生人"

func main() {
    name := "阿莉"
    age := 20
    fmt.Println(defaultGreeting, name)
    fmt.Printf("%s 年龄：%d\n", name, age)
}

运行后，程序会温柔地输出：它是程序世界里最简单的对话。

第 6 节：常见误区（别踩雷）

  • 未使用的变量会导致编译错误——Go 要求每个变量都要被“认真对待”。
  • 短变量声明只能在函数体内使用。
  • 不要把类型和变量名写成同一个词，否则会让人困惑。

第 7 节：从零到一的练习（小任务）

完成如下小练习，将帮助你把知识变成习惯：

  1. 写一个程序，声明一对常量与变量，输出一段话。
  2. 尝试不同类型的变量，体会类型转换。
  3. 在函数内使用短变量声明，体会它与 var 的不同。

结尾 — 再见亦如初见

当你合上编辑器，变量的名字仍像情诗里的押韵字，常量的承诺像月光不再改变。Go 的世界并不冷漠，只要你愿意用心，它会给你真挚的回报。
